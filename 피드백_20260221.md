# 동아·동서 글로컬 연합대학 홈페이지 디자인/인터랙션 피드백

**검토일**: 2026.02.21
**검토 URL**: https://dev-five-git.github.io/donga-dongseo-glocal/
**검토 환경**: Desktop 1440x900 / Mobile 390x844 (iPhone 기준)

---

## 1. 헤더/내비게이션

### 1-1. 메뉴 간격 애니메이션이 부자연스러움
- 스크롤 시 헤더가 등장하면서 메뉴 아이템(혁신방향 / 통합산단 / 연합전공 / Field 캠퍼스) 간 간격이 갑자기 벌어지는 느낌
- 헤더 자체에 `transition: transform 0.3s`가 걸려 있지만, 메뉴 아이템 간 spacing에는 별도 전환 효과가 없어서 "뚝" 끊기는 인상
- 아이템 간 간격이 약 60px 수준으로 넓은데, 상단 헤더에서 로고-메뉴-D-insight 사이의 여백 분배가 고르지 않음. 메뉴 그룹이 중앙에 있으나 로고 쪽으로 치우쳐 보임

### 1-2. 헤더 배경 전환 없음
- 히어로(연보라 배경) → 영상(어두운 배경) → 혁신전략(흰 배경) 등 배경색이 달라지는 구간에서 헤더 배경이 항상 투명(transparent) 고정
- 어두운 배경 위에서 헤더 텍스트(검정)가 잘 안 보이는 구간이 있음
- 스크롤 위치에 따라 헤더에 backdrop-blur나 배경색 전환이 필요

## 2. 히어로 섹션 (NOW STREAMING)

### 2-1. 히어로 → 영상 전환이 명확하지 않음
- 히어로 영역 높이가 뷰포트 2배(1800px)로, 텍스트+영상 → 풀스크린 영상으로 전환됨
- 그런데 이 전환이 단순 스크롤이라 "히어로가 끝나고 새로운 섹션이 시작되는" 느낌이 아니라 같은 영역이 그냥 이어지는 느낌
- 히어로 배경(rgb(241,242,255) 연보라) → 영상 풀스크린(검정) 사이에 fade나 scale 트랜지션이 있으면 섹션 전환이 훨씬 자연스러울 것

### 2-2. SCROLL DOWN 인디케이터
- "SCROLL DOWN" + 화살표가 있지만 bounce 등 반복 애니메이션이 없음
- 정적 텍스트라 사용자 시선을 끌기 어려움. 미묘한 상하 bounce나 fade 반복 애니메이션 권장

### 2-3. 영상 프로그레스 바 (01 — 03)
- 영상 하단의 01/03 슬라이더 UI는 깔끔하나, 영상이 자동 전환될 때 프로그레스 바 애니메이션이 눈에 잘 안 들어옴
- 현재 재생 중인 영상이 어떤 건지 직관적으로 인지하기 어려움

## 3. INNOVATION STRATEGY 섹션

### 3-1. 데스크톱 - 스크롤 스냅 미적용 (핵심 이슈)
- 왼쪽 대학 이미지(sticky 고정) + 오른쪽 혁신방향 콘텐츠(스크롤) 구조는 좋은데, **스크롤 스냅이 없어서** 혁신방향 01과 02 사이 어중간한 위치에서 멈출 수 있음
- 한 번 스크롤에 한 혁신방향씩 정확히 넘어가는 snap 동작 필요
- 시안 의도대로라면 wheel 이벤트를 잡아서 한 섹션씩 전환하거나, `scroll-snap-type: y mandatory` 적용

### 3-2. 데스크톱 - 타임라인(프로그레스) 인디케이터 비활성
- 상단에 "혁신방향 01 / 02 / 03 / 지향점" 가로 타임라인 + 도트가 있음
- 그런데 **스크롤 위치에 따라 활성 상태가 변하지 않음** — 4개 도트가 전부 동일한 회색(rgb(116,116,126))
- 현재 보고 있는 항목의 도트를 브랜드 컬러(#3E31D5)로 전환하고, 타임라인 라인도 진행률에 맞게 색상 채워지는 연출이 필요

### 3-3. 데스크톱 - 왼쪽 배경 이미지 전환 없음
- 왼쪽 sticky 영역이 동일한 대학 건물 사진으로 고정
- 혁신방향 01 → 02 → 03 → 지향점으로 넘어갈 때 배경 이미지가 fade 크로스페이드로 바뀌어야 시각적 변화가 생김
- 현재는 정적 이미지라 오른쪽만 스크롤하는 느낌이 강하고, 양쪽이 하나의 스토리로 연결되는 인상이 약함

### 3-4. 데스크톱 - 콘텐츠 진입 애니메이션 없음
- 혁신방향 카드(이미지 + 텍스트)가 스크롤로 뷰포트에 들어올 때 그냥 "이미 있는 상태"
- fade-in-up 또는 slide-in 같은 등장 애니메이션이 적용되면 각 항목에 주목도가 올라감

### 3-5. 모바일 - 가로 스와이프 미작동 (핵심 이슈)
- 모바일에서 4개 혁신방향이 세로로 나열되어 일반 스크롤로 넘어감
- 하단에 pagination dot(4개)이 있으나 **스크롤과 연동되지 않음** — 항상 같은 dot만 활성
- 의도한 동작: 좌우 터치 스와이프로 한 장씩 넘기는 가로 캐러셀, dot이 현재 슬라이드와 동기화

## 4. FIELD CAMPUS 섹션

### 4-1. 스티키 스크롤 미적용 (핵심 이슈)
- 지도 + 6개 캠퍼스 탭 + 카드가 일반 스크롤로 그냥 지나감
- 의도: 이 섹션이 뷰포트 한 화면에 고정(sticky)되면서, 탭 클릭 시 지도 핀과 하단 카드가 전환되는 인터랙션
- 현재는 B-헤리티지만 활성 상태이고, 다른 탭 클릭 시 카드 전환은 되지만 스크롤로 자연스럽게 넘어가는 연출은 없음

### 4-2. 지도 핀 인터랙션 부족
- 부산 지도 위에 아이콘 핀이 있으나 hover/클릭 시 시각적 피드백이 뚜렷하지 않음
- 현재 활성 캠퍼스 핀이 강조되는 효과(scale-up, pulse, 색상 변경 등)가 있으면 직관적

### 4-3. 캠퍼스 카드 캐러셀
- 데스크톱에서 6개 카드가 가로 나열인데 `overflow: visible` + `scroll-snap: none` 상태
- 카드 간 전환에 snap이나 슬라이드 애니메이션이 없고, 탭 클릭으로만 전환됨
- 카드에 hover 시 미묘한 scale-up이나 shadow 변화가 있으면 인터랙티브 느낌 상승

## 5. CORE ACHIEVEMENTS 섹션 (영상 3개)

### 5-1. 배경색 미적용
- 시안 의도는 보라색(#3E31D5 계열) 배경이었으나, 현재 **흰색/투명** 상태
- 가운데 영상(토론회) 자체가 보라색 톤이라 언뜻 적용된 듯 보이지만, 실제 섹션 배경색이 빠져 있음
- 3개 영상 카드의 통일감을 위해서도 배경색 적용 필요

### 5-2. 스티키/마그네틱 스크롤 미적용 (핵심 이슈)
- 이 섹션이 뷰포트에 진입하면 한 화면에 고정되면서 숫자(15,000+ / 2,000억+ / 7,000+)가 카운트업 애니메이션으로 올라가는 연출이 있어야 임팩트가 있음
- 현재는 그냥 스크롤로 지나가버려서, 핵심 성과 수치가 눈에 안 들어옴

### 5-3. 숫자 카운트업 애니메이션 없음
- 15,000+ / 2,000억+ / 7,000+ 같은 성과 숫자가 정적 텍스트로 렌더링됨
- IntersectionObserver로 뷰포트 진입 시 0에서 목표값까지 카운트업되는 애니메이션이 있으면 임팩트가 확 달라짐

### 5-4. 영상 카드 디자인
- 3개 영상(YouTube 임베드)이 가로로 나열되는데 카드 간 구분이 약함 (간격만 있고 테두리/그림자 없음)
- 카드에 round 처리(현재 0px)나 그림자를 넣으면 개별 카드 인식이 명확해짐

## 6. GLOCAL NOW (뉴스) 섹션

### 6-1. 뉴스 카드 이미지 border-radius 없음
- 뉴스 카드 이미지가 `border-radius: 0px`으로 모서리가 직각
- 사이트 전반의 라운드 처리 톤(12px, 20px, 24px)과 맞지 않음. 이미지에도 라운드 적용 필요

### 6-2. 이미지 object-fit 설정
- 뉴스 이미지가 `object-fit: fill`로 설정되어 비율이 맞지 않으면 이미지가 찌그러질 수 있음
- `object-fit: cover`로 변경해야 비율 유지하면서 영역을 채움

### 6-3. 캐러셀 스크롤 스냅 없음
- 뉴스 카드 컨테이너가 `overflow-x: auto`이지만 `scroll-snap: none`
- 스크롤 시 카드가 어중간한 위치에서 멈춤. `scroll-snap-type: x mandatory` + `scroll-snap-align: start` 적용 필요
- 좌우 화살표 버튼(< >)은 있으나 시각적으로 눈에 잘 안 들어옴 (배경색 rgb(245,247,250)이 본문 배경과 거의 동일)

## 7. FAQ 섹션

### 7-1. 섹션 패딩 불일치
- 다른 메인 섹션(Field Campus, Core Achievements, Glocal Now)은 상하 패딩 120px인데, FAQ만 80px
- 전체 리듬감을 위해 통일하거나, FAQ가 보조 섹션이면 의도적인지 확인 필요

### 7-2. 아코디언 전환 애니메이션
- 아코디언 열기/닫기 동작은 작동하나, 열리는 과정에서 콘텐츠가 자연스럽게 slide-down 되는지 확인 필요
- 높이 변화(height transition)가 부드럽게 적용되어야 함

## 8. 전체 디자인 일관성

### 8-1. 컬러 시스템 - 잘 잡혀 있음
- 브랜드 프라이머리: #3E31D5 (인디고)
- 텍스트 다크: rgb(23,23,25) / 미디엄: rgb(63,63,68) / 라이트: rgb(116,116,126)
- 배경: 흰색 / rgb(245,247,250) 라이트그레이 / rgb(241,242,255) 연보라
- 액센트: rgb(44,163,156) 틸 (INTERVIEW 카테고리에만 사용)
- 전체적으로 컬러 일관성은 양호

### 8-2. 타이포그래피 위계 - 양호
- 섹션 타이틀: 48px / 800 weight (경계를 넘어, 새로운 방향, 캠퍼스를 넘어 등)
- 서브 라벨: 21px / 700 weight (INNOVATION STRATEGY, FIELD CAMPUS 등)
- 영문 카테고리 라벨: 14px / 700 weight / letter-spacing +0.42px (uppercase 느낌으로 잘 구분됨)
- 본문: 16~18px / 400 weight / 음수 letter-spacing으로 한글 가독성 확보
- 위계가 명확하고 일관됨

### 8-3. 라운드 처리 불일치
- 버튼/탭: 24px (pill 형태) — 잘 되어 있음
- 혁신방향 이미지: 20px — 양호
- 뉴스 카드 이미지: **0px** — 전체 톤과 안 맞음
- FAQ 아코디언: 별도 라운드 처리 없음
- 통일 필요

### 8-4. 섹션 간 배경색 흐름
- 히어로(연보라) → 영상(검정) → 혁신전략(흰/라이트그레이) → Field Campus(라이트그레이) → Core Achievements(**흰색 — 보라색이어야 함**) → Glocal Now(흰색) → FAQ(라이트그레이) → 푸터(라이트그레이)
- Core Achievements만 배경색이 빠져서 흐름이 끊김
- 흰색→흰색→라이트그레이가 이어지는 구간(Achievements → Now → FAQ)에서 섹션 구분이 잘 안 됨

## 9. 스크롤 경험 종합

### 9-1. 전체 인상
페이지의 디자인 시스템(컬러, 타이포, 레이아웃)은 잘 잡혀 있으나, **스크롤 인터랙션이 전반적으로 미구현 상태라 "정적인 긴 페이지"로 느껴짐**. 시안에서 의도한 몰입형 스크롤 경험과 현재 구현 사이에 상당한 갭이 존재.

### 9-2. 미구현 인터랙션 요약
| 섹션 | 기대 인터랙션 | 현재 상태 |
|------|-------------|----------|
| 히어로 → 영상 | fade/scale 전환 | 단순 스크롤 |
| INNOVATION STRATEGY (데스크톱) | 스냅 스크롤 + 배경 전환 + 타임라인 연동 | 자유 스크롤, 배경 고정, 타임라인 비활성 |
| INNOVATION STRATEGY (모바일) | 가로 스와이프 캐러셀 | 세로 나열, dot 비연동 |
| FIELD CAMPUS | 스티키 1화면 + 탭/스크롤 전환 | 일반 스크롤 통과 |
| CORE ACHIEVEMENTS | 스티키 + 카운트업 + 보라 배경 | 일반 스크롤, 배경 미적용, 카운트업 없음 |
| GLOCAL NOW 캐러셀 | 스크롤 스냅 | 자유 스크롤, 카드 어중간하게 멈춤 |

### 9-3. 우선 수정 권장 순서
1. **INNOVATION STRATEGY 스냅 스크롤 + 타임라인 연동** (데스크톱) — 페이지의 핵심 스토리텔링 구간
2. **INNOVATION STRATEGY 모바일 가로 캐러셀** — 모바일 UX의 가장 큰 문제
3. **CORE ACHIEVEMENTS 배경색 + 카운트업 + 스티키** — 성과를 어필하는 핵심 섹션
4. **FIELD CAMPUS 스티키 + 전환 인터랙션** — 캠퍼스 소개의 몰입감
5. **헤더 배경 전환 + 메뉴 간격 조정** — 전체적인 내비게이션 완성도
6. **디자인 디테일 (뉴스 카드 라운드, 이미지 비율, 스크롤 스냅 등)** — 마감 품질
